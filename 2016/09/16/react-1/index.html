react深层次的理解

发表于 2016-09-16   |   分类于 react   |     |   阅读次数

一、react与es6

以前写react可以会很繁琐，有了es6就一切都变得很简单了
首先我们从es6对类的支持说起

es5中的类：

function Point(x, y) {
  this.x = x;
  this.y = y;
}

Point.prototype.toString = function () {
  return '(' + this.x + ', ' + this.y + ')';
};

var p = new Point(1, 2);
Object.keys(Point.prototype)    // ["toString"]
Object.getOwnPropertyNames(Point.prototype)   // ["constructor","toString"]

es6中的类：

//定义类
class Point {
  constructor(x, y) {
    this.x = x;
    his.y = y;
}

toString() {
  return '(' + this.x + ', ' + this.y + ')';
}}
Object.keys(Point.prototype)     // []
Object.getOwnPropertyNames(Point.prototype)    // ["constructor","toString"]

不一样的是es6定义的prototype是不可枚举的
当我们将页面组件化的时候，React有提供一个createClass的方法，这可以让我们简单的创建一个组件的类，es6支持类的写法之后，这一切都变得不再繁琐
react的es5表示

var PagePic = React.createClass({
    render:function(){
        return(<img />);
    }
});

react的es6表示

class PagePic extends React.Component {
    render(){
          return(<imgsrc={'https://graph.facebook.com/'+this.props.pagename+'/picture'}/>);
     }
}

继承自React.Component的类与React.createClass是一样的，他们都可以使用React生命周期的函数
React的生命周期指定了React组件渲染过程中处于什么状态就得做什么样的事，这些状态的控制我想React都是已经帮我们控制好了的prototype的函数，原本在es6前使用也是跟render:function(){}一样，但是在有了es6之后可以在class中指定prototype的函数，即使用方法与render(){}一样。

二、react的生命周期


var test = React.createClass({  
    getDefaultProps: function() {  
        console.log("getDefaultProps");  
        return {};  
    },  

    getInitialState: function() {  
        console.log("geyInitialState");  
        return {};  
    },  

    componentWillMount: function() {  
        console.log("componentWillMount");  
    },  

    render: function() {  
        console.log("render");  
        return (  
        <div>hello <strong>{this.props.name}</strong></div>  
        );  
    },  

    componentDidMount: function() {  
        console.log("componentDidMount");  
    },  

    componentWillRecieveProps: function() {  
        console.log("componentWillRecieveProps");  
    },  
    componentWillUpdate: function() {  
        console.log("componentWillUpdate");  
    },  
    componentDidUpdate: function() {  
        console.log("componentDidUpdate");  
    },  

});  

var div1 = ReactDOM.render(  
    <test name='aaa'></test>,  
    document.getElementById("div1")  
);  

var div2 = ReactDOM.render(  
    <test name='bbb'></test>,  
    document.getElementById("div2")  
);

在此大家可以思考一下console.log的打印顺序（如果你觉得你理解react的生命周期的话）
我们创建了一个test类，一个div1对象，一个div2对象
那么在创建类的时候，输出：

getDefaultProps

在创建div1和 div2的时候都会输出：
geyInitialState
componentWillMount
render
componentDidMount

于是执行这段代码，最终输出的就是：

getDefaultProps
getInitialState
componentWillMount
render
componentDidMount
getInitialState
componentWillMount
render
componentDidMount

此时已经存在一个类test和 两个对象 div1 和 div2
然后我们执行 list1.setProps({name: “ccc”})。那么输出如下：

componentWillUpdate
render
componentDidUpdate

可以发现页面上的 hello aaa 已经变成了 hello ccc

生命周期函数的状态

* getInitialState: 获取 this.state 的默认值
* componentWillMount: 在render之前调用此方法，在render之前需要做的事情就在这里处理
* render: 渲染并返回一个虚拟DOM
* componentDidMount: 在render之后，react会使用render返回的虚拟DOM来创建真实DOM，完成之后调用此方法。
* componentWillRecieveProps:   父组件或者通过组件的实例调用 setProps 改变当前组件的 props 时调用。
* shouldComponentUpdate: 是否需要更新，慎用
* componentWillUpdate: 调用 render方之前
* render
* componentDidUpdate: 真实DOM已经完成更新。

三、react的props与state

state是个很神奇的东西，state一发生变化，组件就会重新渲染
state的定义现在我们可以在constructor中进行定义

constructor(props) {
        super(props);
        this.state = {
            xxx: yyy
        };
}

state发生变化，我们可以直接调用react的setState进行改变，酱紫组件也会自动的发生变化
this.setState({xxx:yyy,aaa:bbb});
然而下面引出了另外一个问题，现在组件内发生变化，我们还是很容易的可以通过state进行控制的，但是如果现在一个组件发生变化，另一个组件也要同时发生变化，这个我们改如何进行控制呢？使用redux之类的也是一种方法，但是在小项目里面这个太大材小用了。
我目前的想法是利用props，props是可以在组件与组件之间进行传递的，假如，现在一个组件发生变化，要在另外一个组件体现出来，我们可以找到这两个组件共同的父组件，然后这两个组件都从父组件继承了props，第一个组件props传递的可以是函数，这个函数控制着state，若第一个组件发生了变化，就触发这个函数，使父组件的state发生变化，state发生变化后，第二个组件的props刚好是这个state，props
也随着发生了变化，这会触发第二个组件的componentWillRecieveProps (),第二个组件要作何改变可以把代码写在这个函数里面。
Example：
假设现在有一个项目列表，然后项目列表下面有一个页脚，这个页脚的功能是控制着换页，和显示当前的页数，这个页脚和项目列表同时出现在一个页面上，但是他们又是两个不同的组件，当然这样分配也是因为为了提高组件的利用率，页脚的页数必须与当前的项目列表保持同步，项目列表一次只能显示10条数据，当按下上一页或者下一页的时候，或者直接输入页数的时候，必须同步显示当前的项目列表，这个也涉及到了组件间的通信，实现如下：

export default class UserList extends React.Component{
     constructor(props) {
        super(props);
        this.state = {
            curPage: 1,
            userList: [],
            userPageInfo: {},
            getUserListStatus: ""
        };

        this.getUserList = this.getUserList.bind(this)
      }

      getUserList(curPage){
          this.setState({
              getUserListStatus: 'init'
          });
          Http.get('。。。' + curPage).then((res)=>{
          if(res.success){
              if(res.dataList.length > 0){
                this.setState({
                    userList: res.dataList,
                    userPageInfo: res.pageInfo,
                    getUserListStatus: 'success'
                })
              }else{
                  this.setState({
                    getUserListStatus: 'empty'
                })
              }

          }else{
            console.log(res.resultMsg + '['+ res.resultCode +']')
            this.setState({
                getUserListStatus: 'fail'
            })
          }
        })
      }
     render(){
        var self = this;
        let _list = this.state.userList.map((item, i)=>{
            return(
                <tr key={i}>
                    <td>{item.name}</td>
                    <td>{item.account}</td>
                    <td>{item.email}</td>
                    <td>{item.qqNumber}</td>
                    <td>{item.contact}</td>
                    <td className="oper-box">
                        <Link to={'/userCenter/userDetail/' + item.id} className="oper">详情</Link>
                        <Link to={'/userCenter/editUserInfo/' + item.id} className="oper">修改</Link>
                    </td>
                </tr>
            )
        });
     let result = () =>{
            let status;
            switch(this.state.getUserListStatus)
            {   
                case 'init':
                    status = <tr><td colSpan="6" className="text-center">正在加载中...</td></tr>;
                    break;
                case 'empty':
                    status = <tr><td colSpan="6" className="text-center">暂无数据</td></tr>;
                    break;
                case 'success':
                    status = _list;
                    break;
                case 'fail':
                    status = <tr><td colSpan="6" className="text-center">加载失败</td></tr>;
                    break;
            }

            return status;
        };
     return (
            <div className="UserList-container">
                <h4>用户管理</h4>
                <table className="table table-hover table-bordered table-striped">
                    <thead>
                        <tr>
                            <th>用户名</th>
                            <th>账号</th>
                            <th>邮箱</th>
                            <th>QQ</th>
                            <th>联系方式</th>
                            <th>操作</th>
                        </tr>
                    </thead>
                    <tbody>
                        {result()}
                    </tbody>
                </table>
                <Page
                    totalPage={this.state.userPageInfo.totalPage}
                    selectPage={this.getUserList}
                />
            </div>
        )
}

Page组件的功能大概是上下翻页，有向上点击和向下点击的功能，Page组件需从父组件中知道总页数，获取相应页数数据的函数，这时组件Page就可以进行控制了，当我们点击下一页的时候（Page默认为第一页）然后Page的当前页数加1，Page调用this.props.selectPage(curpage)，于是父组件的项目内容的state发生变化，于是它便重新渲染了，当然我们可以在Page里面设置更多的内容，比如现在我们默认是每个页数10条内容，那Page里面会有个this.perPage=10，但是我们又希望现在不是10条的情况，这样的话可以酱紫写this.perPage=this.props.perPage ||10，需要的话就给Page组件一个props为perPage的属性
那如果table被拆分为另外一个组件怎么办？
这样的话，给table这个组件props.list=this.state.userlist,当props发生变化的时候，table组件会触发componentWillReceiveProps(nextProps)，此刻我们可以将table组件中相应的state设置成相应的，这样组件就会重新渲染了。、
