结果显示在message的div里面两个的file是一样的！！！想了一会，感觉打码还是需要休息的，不然脑会热，搞了个异步加载，就把自己往坑里带，什么叫异步加载，就是不会发生阻塞，在第一个文件上传成功后，第二个文件上传不会受到第一个文件then后面函数的影响，等到第二个上传的时候，当前文件是第二个，post完毕，then开始执行，于是发生了message的div里面两个的file是一样的，而且跟第二个文件是一样的，解决方法如下

processFiles() {
      var message = document.getElementById("message");

到了这里，可能很多人很好奇，Http是怎么封装的，不废话，上代码

 window.Http = {
    ajax: function(o) {
        //Promise是es6新增，访问成功，返回内容resolve(),失败reject()
        return new Promise(function(resolve, reject) {
            var resolved = false;
            //挺喜欢这个缓存的，它可以缓存ajax，加速加载，localStorage是html5新增的web storage
            if (o.cache) {                    
                var cache = localStorage.getItem(o.url);
                if (cache !== null) {
                    resolve(JSON.parse(cache));
                    resolved = true;
                }
            }
            if (!o.method) o.method = 'GET';
            if (typeof o.async === 'undefined') o.async = true;

            var req = new XMLHttpRequest();
            req.open(o.method, o.url, o.async);
            req.overrideMimeType('application/json');
            var data;
            //上传的数据都是json格式
            if (o.data) {
                if (typeof o.data === 'string') {
                    req.setRequestHeader('Content-Type', 'application/json');
                    data = o.data;
                } else {
                    data = new FormData();
                    for (var k in o.data) {
                        data.append(k, o.data[k]);
                    }
                }
            } else {
                data = null;
            }
            req.onreadystatechange = function() {
                if (req.readyState === XMLHttpRequest.DONE && req.status === 200) {
                    if (req.status === 200) {
                        var resp;
                        if (req.responseType !== 'json') {
                            resp = JSON.parse(req.response);
                        } else {
                            resp = req.response;
                        }
                        if (!resolved) {
                            resolve(resp);
                        }
                        //执行回调
                        if (typeof o.callback === 'function') {
                            o.callback(resp);
                        }
                        if (o.cache) {
                            localStorage.setItem(o.url, req.responseText);
                        }
                    }
                }
            };
            req.send(data);
        });
    },
    get: function(url) {
        return window.Http.ajax({
            url: url
        });
    },
    post: function(url, data) {
        return window.Http.ajax({
            url: url,
            method: 'POST',
            data: data,
            contentType: "json",
            dataType: 'json'
        });
    }
};

module.exports = window.Http;

面试的时候，考官问我移动端开发字体大小确定问题还有屏幕分辨率问题，不知从哪方面回答它，但是我脑子里，就感觉以前好像看过这类问题的解决，但是想不起，真是人老了记忆力也衰退了，还好当时做了笔记，与大家共享，其实我想说的是物理像素和逻辑像素的问题，我们开发要考虑的是逻辑像素。
px: css puxels逻辑像素，浏览器使用的抽象单位
dp,pt: device independent pixels设备物理像素
dpr: devicePixelRadio 设备像素缩放比
ppi: 屏幕每英寸的像素书响亮，即单位英寸内的像素密度
计算公式：以iPhone5为例子
ppi=（1136^2+640^2）/4=326ppi（视网膜Retina屏）
注意：单位为硬件像素，非px
PPI越高，像素数越高，图像越清晰
但可视度越低，系统默认设置缩放比越大


或许这个就是答案吧，我的理解是给出设计稿给出的宽高，设备的屏幕只是设备分辨率（物理像素），现在我们要转换成逻辑像素，根据ppi=（w^2+h^2）/4计算出当前单位英寸内的像素密度，可以根据1px = (dpr)^2*dp计算或者对比上面图一找出屏幕默认缩放比dpr，然后，根据缩放比计算出逻辑像素，最终以逻辑像素为标准。写到这里表示没眼看，感觉可能以后自己会再次忘记，没事~手动笔记，多用几次。
字体大小问题，应该想到rem神器，和em（font size of the element）的区别是，em是根据其父元素的字体大小来设置，而rem是根据网页的根元素（html）来设置字体大小的，举一个简单的例子，现在大部分浏览器IE9+，Firefox、Chrome、Safari、Opera ，如果我们不修改相关的字体配置，都是默认显示font-size是16px即

html {
    font-size:16px;        //默认16px！！！
}

那么如果我们想给一个P标签设置12px的字体大小那么用rem来写就是

p {
    font-size: 0.75rem; //12÷16=0.75（rem）
}
